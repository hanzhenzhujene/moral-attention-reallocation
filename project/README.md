# Project Internals

This directory groups implementation and build materials that support the public repo, so the repository root can stay focused on the reader-facing entry points.

## Contents

- [configs/](configs/): canonical public configs and historical execution snapshots
- [paper/](paper/): LaTeX manuscript, paper build notes, and compiled PDF
- [prompts/](prompts/): prompt templates used by the public and historical runs
- [schemas/](schemas/): JSON schemas for benchmark items and run records
- [templates/](templates/): benchmark curation templates
- [requirements.txt](requirements.txt): pinned Python runtime for public reproduction
- [environment.yml](environment.yml): optional Conda environment definition

For most readers, the root [README.md](../README.md), [docs/](../docs/), [results/](../results/), and [scripts/](../scripts/) are the best places to start.
